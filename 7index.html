Branches o que é ?

O branch (ramos) no Git são semelhantes a um ramo de uma árvore, onde o tronco seria a base do código. Desse modo é possível criar diversos ramos e fazer alterações, enquanto a base permanece intacta. O branch principal é denominado de main (master, na versão antiga) permitindo ao desenvolvedor alterar e corrijir a sua pagína com segurança.

Pode-se criar um branch através do branch-padrão do repositório, nele poderá trabalhar nesse novo branch e não irá alterar o branch-padrão.

Merge

É uma união de duas branches, são feitos na branch master.

Clone

Um clone de um repositório funciona como uma branch de um repositório online em um repositório local.

Pull

É uma atualização do repositório local, fazendo um merge do repositório online com o local para que o conflitos sejam resolvidos.

Push

Envia o código para o repositório online

Pull Request

É um pedido que se faz ao dono do repositório para que esse atualize o código dele com o seu código.

Criar um branch

Passo 1
No GitHub, navegue até a página principal do repositório:

Passo 2
Acima da lista de arquivos, clique em Number branches, clique no menu seletor de branch.

Passo 3
Digite um nome exclusivo para o novo branch e selecione Create branch(Criar branch).

Exibir branches no repositório

Use a navegação na parte superior da página para exibir listas específicas de branches:
Your branches
Nessa parte ele mostra todos os branches em que você fez push com os mais recentes primeiro.

Active branches
Nele mostra todos os branches em que algém fez commit nos últimos três meses, ordenados pelos branches com os commits mais recentes primeiro.

Stale branches
Mostra todos os branches em que ninguém fez commits nos últimos três meses, ordenados pelos branches com os commits mais antigos primeiro

All branches
Mostra o branch padrão, seguindo por todos os outros branches, ordenados pelos que têm os commits mais recentes primeiro.

Trabalhando com branches

Quando concluir o seu trabalho, você abrirá uma pull request para fazer merge das alterações do branch atual com outro branch.Depois que a pull request tiver sido mesclada ou fechada, poderá excluir o branch head, já que isso não é mais necessário, para poder excluir você deve ter permissão de gravação.

Hospedando um site para o seu usuário ou organização

Para hospedar um site para o seu usuário ou organização, basta iniciar um repositório de nome username.github.io, onde username é o nome do seu usuário no GitHub ou da sua organização.



Para que o conteúdo seja hospedado diretamente pelo GitHub Pages é necessário que ele esteja no branch master do repositório. Depois de atualizá-lo, com um arquivo index.html por exemplo, já podemos acessá-lo através da URL http://username.github.io.
